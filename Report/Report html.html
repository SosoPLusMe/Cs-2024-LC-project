PLAN AND DESIGN

Basic requirement 1

When researching pokemon sleep I found they record sleep by using a phone's accelerometer and since Micro Bits have an accelerometer I think it might be possible for the MicroBit to emulate this.

These are the inputs that were used for my artefact:

Analog inputs

  • Accelerometer (tilt)

  • Sound sensor


Digital inputs

  • Buttons on MicroBit

  • Microbit logo


These are the outputs:

Analog Outputs

  • The amount of times you woke up.

  • Noise Level.

  • Sleep duration.

  • How long MicroBit was on.

  • Quality of sleep.

  • Stress level.

  • Serial output


Digital Outputs

  • LEDs on MicroBit

  • Beeping from speakers


With my artefact you can start sleep sessions by pressing button ‘A’ and end it by pressing buttons ‘A’+’B’. During a session if you get up and this adds to a counter and the noise level is also taken at regular intervals.


When you end the session you are able to input your quality of sleep and stress level. After inputting these the MicroBit will then serial out the following information:


  • The amount of times you woke up.

  • Noise Level.

  • Sleep duration.

  • How longMicroBit was on.

  • Quality of sleep.

  • Stress level.


Basic requirement 2

After this, python code cleans and stores the data into a file in a csv format. The data was validated in the MicroBit code.


Basic requirement 3

An analysis component is created with python that can be used to calculate and predict the amount of times a user would wake up based on the noise level whilst they sleep. The python code also informs the user on future decisions relating to sleep based on the predicted noise level. The advice given comes from this website. https://www.everest.co.uk/windows/noise-reduction-window/how-to-soundproof-room/


Advanced requirement 1

Using python with the libraries Matplotlib, ScikitLearn and Pandas I was able to train and create a model. The model uses three parameters from a database these parameters being the noise level, the stress level of the user and the amount of times they got up. When given these parameters the model is able to predict the quality of sleep the user had.

The features of wellbeing this relates to are

  1. Reducing stress and anxiety

  2. Monitoring health

  3. Enhancing sleep quality.


Advanced requirement 2

The three what if questions devised by me are:

  1. What if all three parameters are low?

  2. What if all three parameters are high?

  3. What if one parameter is high whilst the rest are low?


My model was able to answer these questions showing that when all parameters are low the quality of sleep is higher than if all the parameters were high. And when answering the third question it gives the user the wellbeing insight that the stress level is the biggest indicator of quality of sleep which I think makes sense from a wellbeing standpoint.

  • This is an important piece of code for my MicroBit. I will explain what this code does.

When button ‘A’ is pressed a variable ‘start’ is set to the running time this will be subtracted from the running time at a later point and will be used to know how long the user has slept for.


While the process is true (it is set to true on startup), it will add the noise level to the average an LED is shown to indicate the process is working.


After 2 seconds the LED is turned off and if ‘A’+’B’ has been pressed, ‘process’ will be set to false and it will exit the loop. But if ‘A’+’B’ have not been pressed then if the pitch value is over 45 or under -45(the user is upright during these pitches) then it will raise the ‘Isup’ value by one. The Isup value is used to see how many times the user got up during the night


While the user is upright then it plays a sound to remind the user to log their data if they have woken up and are not going back to sleep and the volume is played at a low volume to not disturb the user if they are planning to go back to sleep. This whole process repeats until the user presses buttons ‘A’ + ‘B’.

EVALUATION

Looking back at this project I see that it has met the requirements set out in the brief by creating a fully automated embedded sleep monitor that utilised digital (sound sensor and tilt sensor) and analogue (buttons) inputs and

Digital(serial out) and analogue(beeping noise and LEDs) outputs to support a theme of wellbeing that being ‘monitoring sleep’.

I validated and stored the data from the MicroBit and created an analysis component that could be used to calculate the noise level based on the amount of times a user awoke.

And for the advanced requirements using Python I created a computer model based on my own personally created dataset that contained descriptive wellbeing data such as the Stress level after sleep and Quality of sleep depending on noise level.

The model was capable of answering three ‘what if’ type questions which I devised myself.


Each ‘what if’ question used a minimum of three validated parameters (using at least two

different data types(int + float)) and, based on the information provided, offered the user insights in relation to some aspect of their wellbeing.

Users can view the results of the ‘what if’ scenarios in graphical form.


In relation to the needs of the end user(s)The artefact does show how noise levels impact their sleep, the length of their sleep and is active in monitoring by beeping when they are up and having a small flashing light when they are lying down. It also gives advice on reducing noise level and insights into their wellbeing.


If I was to revisit this project I would like to improve how the artefact knows the user is awake because as it is now it considers lying down to be asleep. Maybe if there is a difference between a person's heart rate when they are asleep and when they are awake I could base the artefact off the users heart rate and not not position thus making it much more reliable. The artefact itself also does need to be connected to a computer which has a negative impact on a person's sleep so if it was wireless and also logged the data wirelessly to the computer it would be a huge improvement.


All in all I am proud of this project and this report. I thank you for reading.
